Appendix

Glossary

Every term the book introduces, in one line each, with where it first appears.

The stack

TermMeaningCh
APIA menu of functions one program offers another. "Ask me this way, get an answer in this shape."2
XInputWindows' API for Xbox-style controllers. Games call XInputGetState; it supports exactly four pads.2
DirectInputThe older, more general Windows input API. Clumsier with triggers. Mostly superseded.2
HIDThe USB standard for input devices generally — the layer beneath both input APIs.2
DriverSoftware letting the OS talk to one kind of hardware. Runs privileged, in kernel mode.2
Kernel modeThe OS's inner circle — unrestricted access to hardware. Where drivers live.2
User modeWhere ordinary programs run, walled off from hardware. Where your server lives.2
Virtual bus driverA driver that reports devices on a bus that exists only in software. The core trick.3
ViGEmBusThe specific virtual bus driver we use. Emulates Xbox 360 and DualShock 4. Retired Nov 2023, still works.3
ViGEmClientThe user-mode library your program calls to talk to ViGEmBus. Not the same thing as the driver.3
ScpVBusViGEmBus's predecessor, by partly the same author. What DroidJoy is built on.12
uinputLinux's built-in equivalent — lets user-mode programs create input devices with no custom driver.3
XINPUT_GAMEPADThe 12-byte structure holding a controller's entire state.4

Input

TermMeaningCh
State vs eventsA complete snapshot ("A is down") versus a change notification ("A was pressed"). We send state. Everything follows from this.1
DigitalTwo values only — on or off. Buttons and keyboard keys.1
AnalogA continuous range. Sticks and triggers. The reason fake keystrokes aren't good enough.1
BitmaskMany yes/no answers packed into one number, one bit each. All 16 buttons in a single 16-bit value.4
PointerAndroid's name for one finger on screen. Has a stable ID and an unstable index — always use the ID.9
DeadzoneA centre region reported as zero. Do it radially, or diagonals snap to the compass points.9
Floating originThe stick centres wherever your thumb lands, rather than at a fixed spot. Biggest feel win for the least code.9

Networking

TermMeaningCh
Client / serverThe server waits at a known address; the client goes looking. Nothing to do with which is more powerful.1
IP addressIdentifies a machine. 192.168.… addresses are private to your network and can't be reached from outside.5
PortIdentifies which program on that machine gets the data. Building, then flat number.5
SocketYour program's phone line. Create, bind, send, receive.5
127.0.0.1"This same machine." Lets you test without any network involved.5
TCPGuarantees delivery and order — by stalling everything when data is lost. Wrong for realtime input.7
UDPGuarantees nothing, never waits. Right for us, because stale data is worthless anyway.7
Sequence numberA per-packet counter so the receiver can discard anything older than what it has.7
ProtocolAn agreement about what the bytes mean. Mismatches fail silently, not loudly.8
EndiannessWhich byte of a multi-byte number comes first. Pick little-endian; Kotlin's ByteBuffer defaults the other way.8
Broadcast255.255.255.255 — "everyone on this network." UDP only, and can't leave your LAN.10
mDNS / DNS-SDThe standards-based discovery mechanism printers and Chromecasts use. Tier 3.10
AP isolationA router setting stopping wireless devices from reaching each other. Defeats broadcast; you cannot work around it.10
adb reverseTunnels a port over the USB cable. Lower and steadier latency than Wi-Fi. Your best chance to beat the competition.12

Performance & judgement

TermMeaningCh
MillisecondA thousandth of a second. A 60 Hz frame is 16.7 ms. Aim for under ~50 ms end to end.6
LatencyHow long from thumb to pixel. Roughly 34 ms typical — of which your code is about 0.2 ms.6
JitterVariation in latency. Hurts more than a slightly higher constant delay, because hands adapt to consistency.6
Polling rateHow often state is sent. 125 Hz (every 8 ms) matches a real USB gamepad.6
Threat modelAn honest account of who might attack, what they'd gain, and what it'd cost. Keeps security effort proportionate.10
Steam InputSteam's own remapping layer, sitting on top of the OS. Disable it before debugging anything input-related.11
The six ideas worth keeping
  1. Find the layer where the lie is cheapest.
  2. Measure before optimising — effort follows visibility, not importance.
  3. Send state, not events. Self-sufficient messages make loss a non-event.
  4. Do the risky part first.
  5. Trade-offs are priced, not free. Choose deliberately and accept the bill.
  6. Know when to stop. A finished project beats an abandoned ambitious one.
